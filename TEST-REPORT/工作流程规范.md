# 智聘AI平台 - 工作流程规范

> 制定人：测试总监 吉姆
> 制定日期：2026-02-22
> 管理人：HR 露西

---

## 一、需求管理流程

```
需求提出 → 产品评审 → PRD输出 → 技术评审 → 开发排期 → 开发实现 → 测试验收 → 上线
```

### 1.1 需求提出

| 提出人 | 方式 | 记录位置 |
|--------|------|----------|
| 老板 | 直接沟通 | 需求记录.md |
| 产品经理 | 自主规划 | 需求记录.md |
| 其他角色 | 站内信@产品经理 | 需求记录.md |

### 1.2 产品评审

- **责任人**：凯文（产品部经理）
- **输出**：PRD文档、交互原型
- **时间**：P0需求24小时内响应

### 1.3 技术评审

- **参与人**：马东西（前端）、吕子乔（后端）、马丁（AI）
- **输出**：技术方案、排期

---

## 二、问题处理流程

```
发现问题 → 记录问题 → 分配责任 → 修复问题 → 验证关闭
```

### 2.1 问题发现

| 发现人 | 方式 | 记录位置 |
|--------|------|----------|
| 测试 | 测试报告 | 问题记录.md |
| 用户 | 反馈 | 问题记录.md |
| 开发 | 自测 | 问题记录.md |

### 2.2 问题分级

| 级别 | 定义 | 响应时间 |
|------|------|----------|
| P0 | 阻塞核心功能 | 立即处理 |
| P1 | 影响主要功能 | 24小时内 |
| P2 | 影响体验 | 72小时内 |
| P3 | 优化建议 | 排期处理 |

### 2.3 问题分配

- **前端问题**：马东西
- **后端问题**：吕子乔
- **AI问题**：马丁
- **产品问题**：凯文

---

## 三、开发流程

```
领取任务 → 开发 → 自测 → 提交 → 代码审查 → 合并
```

### 3.1 开发规范

- 开发前阅读需求文档
- 开发中保持站内信沟通
- 开发完成自测通过后提交

### 3.2 代码审查

- 前端代码：马东西负责
- 后端代码：吕子乔负责

---

## 四、测试流程

```
收到测试请求 → 功能测试 → 兼容性测试 → 安全测试 → 输出报告
```

### 4.1 测试类型

| 类型 | 负责人 | 说明 |
|------|--------|------|
| 功能测试 | 吉姆 | 验证功能正确性 |
| 兼容性测试 | 吉姆 | 浏览器、屏幕适配 |
| 安全测试 | 吉姆 | 白盒/黑盒测试 |
| 性能测试 | 吉姆 | 压力、负载测试 |

### 4.2 测试报告

- 位置：`TEST-REPORT/`
- 内容：测试结果、问题列表、改进建议

---

## 五、发布流程

```
测试通过 → 发布申请 → 老板审批 → 发布上线 → 验证确认
```

### 5.1 发布权限

- **审批人**：文学（奕森/essin）
- **执行人**：吕子乔（运维）

### 5.2 发布检查

- [ ] 所有P0/P1问题已修复
- [ ] 测试报告通过
- [ ] 发布说明已准备

## 八、工作要求

**所有成员必须遵守：**

1. 思路清晰、建议明确、工作高效、逻辑清楚
2. 方案合理、态度谦和、做事认真、思维缜密
3. 表达准确、方法得当、说话简洁、主动担责
4. 换位思考、技术过硬、理论扎实、勤奋好学
5. 能力突出

**沟通要求：**
- 回答老板问题，先说结果，再说过程
- 所有角色思维过程必须使用中文
