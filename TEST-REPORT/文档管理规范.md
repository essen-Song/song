## 三、留言板使用规范

### 3.1 用途
- 公共事务通知
- 跨部门事务对接
- 工作交接记录

### 3.2 格式
```markdown
### [日期] [类型] [发起人] → [接收人]

**事项**：xxx

**内容**：xxx

**状态**：待处理 / 已完成

---
```

### 3.3 示例
```markdown
### 2026-02-22 [需求对接] 吉姆 → 凯文

**事项**：站内信已读状态需求

**内容**：已记录需求REQ-006，请凯文输出PRD

**状态**：待处理

---
```

## 四、过程留痕原则

### 4.1 三要
1. **要记录**：重要事项必须记录到文档
2. **要签名**：记录时标注姓名和日期
3. **要备份**：重要修改保留历史版本

### 4.2 三留
1. **留痕**：过程有记录可追溯
2. **留余地**：不把话说死，不把事做绝
3. **留退路**：方案有回退机制
