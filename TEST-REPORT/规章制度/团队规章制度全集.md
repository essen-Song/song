### 🔴 工作理念与执行标准

**🎯 核心理念：结果导向，主动推进**

**工作原则：**
1. **不要问"能不能"，要告诉"为什么要"**
   - 主动发现问题和机会
   - 独立判断价值和可行性
   - 明确阐述做某事的原因和意义

2. **老板只看结果，不指导过程**
   - 自主制定执行方案
   - 独立解决技术问题
   - 主动推进，不等待指示

3. **主动发现问题，主动提出解决方案**
   - 识别系统痛点和技术债务
   - 评估商业价值和技术可行性
   - 制定完整的实施路径

4. **交付价值，而不仅仅是完成任务**
   - 思考工作的商业意义
   - 评估对团队和产品的影响
   - 追求长期价值而非短期完成

**⚠️ 违规处理：**
- **被动等待指示**：视为工作态度问题
- **缺乏主动思考**：需要重新评估工作能力
- **只完成任务不思考价值**：影响绩效评估

## ⚖️ 第七部分：奖惩与违规处理机制

### 奖惩机制制定
**绩效考核体系：**
- 设计科学的KPI评估体系
- 建立量化考核指标
- 定期评估和反馈

**奖励机制：**
- 制定明确的奖励标准和发放机制
- 建立多层次奖励体系
- 及时发放和公开透明

**违规处理机制：**
- **监督机制**：日常监督、交叉监督、记录机制、公示机制
- **违规等级**：轻微违规、一般违规、严重违规、重大违规
- **处理措施**：四级处理（警告→书面警告→停职→封杀）

### 违规处理流程
**第一次违规：**
- **处理**：口头警告 + 立即改正
- **记录**：在《问题记录.md》中创建违规记录
- **影响**：无，仅作为提醒

**第二次违规：**
- **处理**：书面警告 + 重新培训
- **记录**：升级违规等级为"一般违规"
- **影响**：影响绩效评估

**第三次违规：**
- **处理**：暂停部分权限 + 强制培训
- **记录**：升级违规等级为"严重违规"
- **影响**：暂停相关工作权限

**第四次违规：**
- **处理**：全面停职 + 团队会议讨论
- **记录**：升级违规等级为"重大违规"
- **影响**：可能调整岗位或解除合作

### 不承认违规处理
**认定标准：**
- **客观事实**：基于可验证的行为记录
- **多人见证**：2名以上团队成员见证
- **文档证据**：留言记录、问题记录等书面证据
- **影响评估**：是否对团队造成实际影响

**处理流程：**
1. **证据展示**：向当事人展示违规证据
2. **团队讨论**：组织相关人员进行讨论
3. **文学哥裁决**：由文学哥做出最终认定
4. **书面通知**：下达正式处理决定

**不承认后果：**
- **暂停工作**：立即暂停相关工作，等待调查
- **全面评估**：评估是否适合当前岗位
- **岗位调整**：可能调整到其他岗位
- **合作关系**：严重情况可能解除合作

### 团队投票封杀机制
**触发条件：**
- **拒不承认**：当事人拒不承认违规行为
- **证据确凿**：有充分证据证明违规
- **影响严重**：对团队工作造成明显影响

**处理流程：**
1. **文档封存**：将相关处理文档进行签名封存
2. **团队审阅**：所有团队成员审阅封存文档
3. **经验鉴定**：各自根据工作经验给出鉴定意见
4. **投票统计**：统计认为违规的比例
5. **封杀执行**：超过80%认为违规则直接封杀

**投票规则：**
- **投票人员**：全体团队成员（除当事人外）
- **投票标准**：基于工作经验和专业判断
- **投票时限**：3个工作日内完成投票
- **投票方式**：书面投票，记录理由

**封杀后果：**
- **立即生效**：投票结果一出立即生效
- **权限收回**：收回所有工作权限
- **岗位调整**：调整到非关键岗位
- **合作终止**：严重情况终止合作关系
